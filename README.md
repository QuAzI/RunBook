Есть ряд инструментов, совмещающих документацию и блоки команд для быстрого исполнения.

Начиная с "блокнотов" типа Jupyter Notebooks (JupyterLab) и Polyglot Notebooks и заканчивая [tldr++](https://github.com/isacikgoz/tldr).

tldr++ хоть и прекратила своё развитие, но по уровню удобства работы в консоли я пожалуй не видел ничего лучше.

Основные возможности:
- быстрый нечёткий поиск в духе fzf (достаточно помнить частичное написание)
- быстрый запуск команд
- подстановка параметров запуска

tldr++ позволяет не только быстро искать документацию по Markdown (в том числе по описанию, сама база знаний хранится в ~/AppData/Roaming/tldr), 
но и запускать выборочно снипеты команд из документации.
Причём если команда подразумевает ввод параметров, то он и про параметры напомнит и даже предложит имена файлов подставить.

Лучший пример интерактивной справки в моей жизни. К сожалению другие форки tldr так не умеют и быстро наскриптовать что-то аналогичное для PowerShell не получилось (см. fff.ps1).

Но концепция коллекции часто используемых runbooks определённо цепляет и регулярно напоминает о себе.

Хочется иметь библиотеку часто используемых скриптов с описанием, чтобы не вспоминать каждый раз подробности и параметры, 
и инструмент для работы с этой библиотекой в стиле tldr++ с быстрым гибким поиском (на манер fzf)

## Другие полезные инструменты

### PowerShell

Актуальный [PowerShell v7+](https://github.com/PowerShell/PowerShell/releases/) в паре с [Windows Terminal](https://github.com/microsoft/terminal/releases)
дают довольно неплохую основу для работы в консоли. Штатный поиск по истории команд, который раньше был ограничен только стрелками, дополняется теперь
- F8 - ищет предыдущую команду которая начинается на уже введённый текст (набираем `docker` и погнали листать что было)
- F2 - список команд, учитывает уже введённый текст и по нему выводит несколько последних

### Bash

Имеет аналогичные возможности (стрелки для перехода по истории, Ctrl+R для поиска по истории), 
а интеграция с fzf bash completion добавляет возможностей быстрой навигации по файлам и поддиректориям.

### VSCode

В VSCode запустить Command Palette (Cmd+Shift+P) и выбрать
`Terminal: Run Selected Text in Active Terminal`

Либо использовать расширения для быстрого запуска.

### JetBrains IDE / Rider / DataGrip

IDE JetBrains автоматически пытаются определять исполняемые блоки в .md файлах.
Для таких блоков появляется кнопка запуска (Play), либо нужно перейти курсором в блок и нажать Ctrl+Shift+F10.

Точно так же можно запускать запросы в SQL-скриптах (предварительно выбрав БД и схему)
